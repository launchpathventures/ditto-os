# Insight-121: Day Zero and Empty State Are Different Moments

**Date:** 2026-03-31
**Trigger:** Overlap analysis between Brief 057 (first-run experience) and planned Brief 066 (conversation polish layer)
**Layers affected:** L6 Human
**Status:** active

## The Insight

Brief 057's Day Zero page and the conversation empty state serve different purposes and must not repeat each other. Day Zero is the introduction — "what makes Ditto different" (shown once, never again). The conversation empty state is the workspace prompt — "what should we work on?" (shown every new conversation).

Brief 057 already solves the sequencing: Self speaks first for new users (AC14), so the conversation is never empty on first arrival. The user sees Day Zero → clicks through → lands in workspace with Self's welcome message already present. The empty state only appears on subsequent new conversations.

## Implications

- Brief 066's empty state heading must NOT repeat "Hi. I'm Ditto." — that's the Day Zero moment
- Use a returning-user greeting: "What would you like to work on?" or contextual suggestions based on existing processes
- The "Hi. I'm Ditto." + dot particles combination is reserved for the one-time Day Zero page
- Suggestion chips in the empty state should be contextual (returning user has processes → "What needs my attention?", "Check on [process]")
