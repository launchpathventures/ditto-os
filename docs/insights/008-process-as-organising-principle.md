# Insight-008: Process as the Organising Principle of the AI-Enabled Organisation

**Date:** 2026-03-19
**Trigger:** Dev PM triage session — user shared strategic vision for how Agent OS relates to organisational design
**Layers affected:** L1 Process, L3 Harness, L4 Awareness, L6 Human
**Status:** active

## The Insight

Processes are the atomic unit of the organisation of the future. The goal is end-to-end processes that produce real-world outcomes valued by individuals, teams, companies, and customers — not automation of isolated steps.

Two dimensions shape process complexity:

1. **Narrow vs wide.** Narrow processes (customer service, sales) are more tractable — a robust agent can deliver them. Wide processes (onboard a new supplier to the supply chain) cut across organisational functions and are fundamentally harder.

2. **Single-agent vs multi-team.** Most valuable end-to-end processes in organisations today span multiple functions. Agent OS must consider how these processes are governed by teams or chains of agents, not just a single agent. Trust, review, and governance all become multi-stakeholder problems for cross-functional processes.

Agent OS is the stem cell for the AI-enabled individual, team, and organisation. It is the ultimate representation of applied AI — the organising substrate for entities that organise around process outcomes rather than departmental boundaries.

## Implications

- **Trust scope (Phase 3):** Trust earning must be designed with multi-stakeholder governance in mind, even if Phase 3 only implements single-agent trust. A narrow process might earn autonomous trust quickly (high volume, clear success criteria). A cross-functional process might stay supervised much longer.
- **Process design (L1):** Process definitions should accommodate cross-functional ownership — steps owned by different teams or agents with different trust levels.
- **Governance (Phase 11):** The governance model needs to handle processes that span organisational boundaries, not just single-team processes.
- **Positioning:** Agent OS is not a task runner or an agent framework. It's the substrate for process-organised work. This affects how we talk about it, what we prioritise, and what we defer.

## Where It Should Land

Architecture spec — should inform the L1 Process layer definition and the Governance section. The narrow-vs-wide dimension could become a first-class property of process definitions (complexity tier or scope indicator). The multi-stakeholder governance model needs to be addressed before Phase 11 — at minimum as a design constraint in earlier phases.
