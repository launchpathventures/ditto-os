# Insight 128: Brand Presence as Interaction Affordance

**Date:** 2026-04-01
**Source:** Prompt input design session — DotParticles → Send button crossfade
**Status:** Active
**Absorbed into:** —

## Observation

The DotParticles (Ditto's brand identity element) were initially placed in the toolbar alongside action buttons, where they felt decorative and "off." Moving them to the send button position — and having them crossfade into the send button when the user types — transforms them from decoration into interaction affordance.

## Principle

**Brand elements should occupy functional positions, not decorative ones.** When a brand element (logo, particles, animation) sits alongside action controls without serving a function, it feels off. When it occupies a functional position and transforms into the action, it creates a distinctive brand moment that competitors can't replicate.

## Application

- The DotParticles sit in the send-button position when idle — Ditto's living presence
- When text is entered, dots fade out and the emerald send button fades in — the presence *becomes* the action
- When streaming ends and input clears, the send button fades out and dots return
- This pattern can extend: dots could appear in other "waiting" positions (loading states, empty states) and transform into functional controls when the user acts

## Why This Matters

Every AI chat app has a send button. None have their brand identity *become* the send button. This is the kind of distinctiveness that `.impeccable.md` demands — bold identity that's functional, not ornamental.
