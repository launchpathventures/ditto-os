# Impeccable Design Context — Ditto

This file persists Ditto's design DNA for all AI-assisted design and build work. Every `/frontend-design` invocation reads this first.

## Design Context

### Users

Ditto serves **outcome owners** — people who own business results but aren't developers. They need AI to handle execution so they can focus on judgment.

**Primary personas:**
- **Rob** — Trades MD, 8-15 staff, mobile between job sites. Approves quotes from his truck. Needs: glanceable, one-tap actions, nothing that feels like admin software.
- **Lisa** — Ecommerce MD, 10-20 staff. Manages content, customer service, warehouse ops. Needs: content quality consistency, proactive issue detection, confidence that things are running well.
- **Jordan** — Generalist technologist in 50-200 person org. Tech-savvy non-developer. Needs: prove value in 48 hours, show leadership the system map, be the leverage point.
- **Nadia** — Team manager, 5-10 specialists. Knowledge work requiring judgment. Needs: team's operational burden handled, focus on coaching not formatting.
- **Libby** — Doula transitioning to online education business. Building phase. Needs: knowledge capture, brand voice, social content creation. Primary prototype persona.

**Context of use:** These people enter work (questions, tasks, goals) into a living workspace. They are NOT managing automation flows or configuring pipelines. Ditto is where they work, not where they administer.

**Emotional journey:** Cautious Hope (Week 1) → Growing Trust (Month 1) → Confident Delegation (Month 2) → Compound Effect (Month 3+). The UI must support this arc — starting simple and intimate, growing structured and powerful.

### Brand Personality

**Three words:** Confident. Warm. Distinctive.

**Voice:** Like a competent colleague who remembers everything, surfaces what matters, and never makes you feel dumb. Not an assistant (too subservient), not a platform (too cold), not a chatbot (too casual).

**Visual identity philosophy:** Inspired by flatwhiteorfckoff.com — do a few things exceptionally well, do the opposite of the competition, find white space. Every AI tool uses dark sidebar + white chat + blue/purple accents + Inter. Ditto doesn't.

**Emotional goals:**
- **Confidence** — "I can see what's happening and trust what I'm seeing"
- **Calm** — "Nothing is screaming at me; the important things surface naturally"
- **Competence** — "This system knows what it's doing"
- **Continuity** — "It remembers. It learns. It doesn't start from scratch."

**Anti-goals:**
- Never feel like Monday.com, Linear, or Notion (passive containers for tasks)
- Never feel clinical or enterprise (no dense data tables, no grey-on-grey dashboards)
- Never feel like a chatbot wrapper (artefacts are primary, chat is secondary)
- Never feel noisy (oversight is quiet, trust-calibrated, exception-driven)
- Never feel like a Claude/ChatGPT clone (no terracotta, no purple, no generic AI aesthetic)

### Aesthetic Direction

**Visual tone:** Confident and warm with a bold signature. Spacious, not compact. Quiet, not minimal. Human, not clinical. Distinctive, not derivative.

**References (what to channel):**
- flatwhiteorfckoff.com — aggressive simplicity, bold identity that cuts through noise
- OpenAI — black/white foundation with splashes of vibrancy
- Claude.ai — conversational layout, generous whitespace, centred content
- Organic growth motif (intertwining lines, nodes) — nature meets technology
- Ditto distinguishes with a **two-green signature** (forest + emerald), **organic typography** (DM Sans), and **living dot particles**

**Anti-references (what to avoid):**
- Retool / internal tools aesthetic (dense, utilitarian, grey)
- Slack (noisy, notification-heavy, overwhelming)
- Enterprise dashboards (charts everywhere, tiny text, no breathing room)
- Generic AI chat apps (dark sidebar, white chat panel, blue/purple accents, Inter font)
- Claude's terracotta — explicitly rejected as derivative

**Theme:** Light mode primary (cool grey canvas). Dark mode supported — forest green (`#3D5A48`) as dark background, white text, emerald accents preserved. Two themes available: **Signature** (green accents) and **Focus** (B&W, no color distractions).

**Colors to avoid:** Terracotta/orange-brown (Claude), purple/violet (generic AI), blue (ubiquitous in tech). Red only for semantic negative/error states.

### Design Principles

1. **Artefacts are primary, conversation is secondary.** The workspace shows what was produced, what needs review, what's changing. Chat is the seed that creates artefacts, not the primary surface. (Insight-080)

2. **Progressive disclosure, not progressive complexity.** Start with the essential. Reveal depth on interaction. The user should never see more than they need at any moment — but everything should be reachable. "Boiling frog" — complexity grows so gradually the user never notices. (human-layer.md)

3. **User language, not system language.** No "Goal," "Trust tier," "Process," "Agent" in labels. Use the words the user would use: "How it's going," "What needs you," "What I know about your business." (Insight-073)

4. **Silence is a feature.** When things are running well, the UI is calm. Autonomous processes appear as one-line digests, not item-by-item noise. The workspace gets quieter as trust grows. (ADR-011)

5. **Every data point has provenance.** Outputs show what knowledge was used ("based on"). Decisions show what evidence supported them. Trust shows what earned it. Progressive disclosure: summary → detail on click. (Insight-087)

6. **No cards. Content flows typographically.** No box containers wrapping every piece of content. Hierarchy is created through typography, whitespace, and subtle accents (border-left lines, surface backgrounds for hero moments only). (P08a v6 design evolution)

7. **Bold identity, not decoration.** The two-green palette (forest + emerald), the dot particle system, and DM Sans are signature elements — not ornamental. They appear everywhere consistently, creating instant brand recognition. The visual metaphor is *organic growth* — a living system, not a manufactured tool.

**Text:**
| Token | Light | Dark | Purpose |
|-------|-------|------|---------|

**Accent — Black (UI controls):**

**Vivid — Brand Signature (Two-Green System):**
| Token | Light | Dark | Purpose |
|-------|-------|------|---------|
| `vivid` | `#059669` | `#10B981` | Emerald — CTAs, dot particles, active states, energy moments |
| `vivid-deep` | `#3D5A48` | `#6B8F71` | Forest green — text accents, borders, brand surfaces, emphasis |
| `vivid-subtle` | `#ECFDF5` | `#1A2E22` | Subtle green backgrounds |

**Rationale:** Forest grounds, emerald sparks. Same hue family (~155°), different saturation (19% vs 97%). Forest passes 4.7:1 on grey canvas for text; emerald at 3.3:1 is reserved for large surfaces/buttons where 3:1 suffices.

**Semantic:**

|-------|-------|------|---------|
| `positive` | `#16a34a` | `#22c55e` | Healthy, approved |
| `caution` | `#D4960A` | `#E5A820` | Warnings, suggestions |
| `negative` | `#C4352A` | `#E8453A` | Errors, failures |
| `info` | `#2563eb` | `#3B82F6` | Informational |

### Background Treatment

Clean cool grey canvas. No ribbon, no background patterns, no gradients. The brand identity lives in **color accents and the dot particle system**, not in background decoration. The grey canvas provides a calm, professional foundation that lets content breathe.

### Typography

| Scale | Size | Line Height | Weight | Usage |
|-------|------|-------------|--------|-------|
| `text-xs` | 12px | 1.5 | Regular | Timestamps, metadata |
| `text-sm` | 14px | 1.5 | Regular | Secondary text, sidebar |
| `text-base` | 16px | 1.6 | Regular | Body, conversation, feed |
| `text-lg` | 20px | 1.4 | Medium 500 | Section headers |
| `text-xl` | 25px | 1.3 | Semibold 600 | Page titles, Self greeting |
| `text-2xl` | 31px | 1.2 | Semibold 600 | Hero text (rare — first greeting only) |

- **Primary font:** DM Sans (sans-serif) — organic, readable, distinctive vs Inter/SF Pro
- **Mono font:** JetBrains Mono (technical detail only — never in user-facing surfaces)
- **Scale ratio:** 1.25 (Major Third)
- **Base:** 16px

### Spacing

4px grid. Generous, not compact.

| Token | Value | Usage |
|-------|-------|-------|
| `spacing-1` | 4px | Icon-to-text gaps |
| `spacing-2` | 8px | Tight element spacing |
| `spacing-3` | 12px | Standard internal padding |
| `spacing-4` | 16px | Card padding, input padding |
| `spacing-5` | 20px | Between elements |
| `spacing-6` | 24px | Between sections |
| `spacing-8` | 32px | Section separation |
| `spacing-10` | 40px | Major section separation |
| `spacing-12` | 48px | Page-level breathing room |

### Border Radius

| Element | Value | Tailwind | Feel |
|---------|-------|----------|------|
| Buttons | 9999px | `rounded-full` | Pill-shaped — bold, distinctive |
| Cards | N/A | N/A | Cards are killed — no box containers |
| Inputs | 24px | `rounded-3xl` | Capsule shape, expands vertically |
| Avatars / Self | 9999px | `rounded-full` | Circular |
| Modals | 16px | `rounded-2xl` | Softer |
| Tooltips | 6px | `rounded-md` | Subtle |
| Surface containers | 12px | `rounded-xl` | For hero moments only (process card, connect card) |

### Shadows

Cool-tinted (rgba 26,26,26), not warm:
- `subtle`: `0 1px 3px rgba(26,26,26,0.05)` — hover states
- `medium`: `0 4px 16px rgba(26,26,26,0.07)` — popovers, dropdowns
- `large`: `0 8px 24px rgba(26,26,26,0.12)` — modals

### Motion

Responsive, not performative. Every animation serves a purpose.

| Type | Duration | Easing |
|------|----------|--------|
| Micro (hover, press) | 100-150ms | `ease-out` |
| Standard (panels, cards) | 200-250ms | `ease-in-out` |
| Entrance | 300ms | `ease-out` |
| Exit | 200ms | `ease-in` |

**Never exceeds 400ms.** No bouncing, spinning loaders, parallax, confetti, or blocking animations. Respect `prefers-reduced-motion`.

### Icons

Lucide React. 20px default, 16px in compact contexts. `stroke-width: 1.5`. Icons accompany text — never standalone without a label (except universally understood: close, menu, search).

### The Self (Ditto's Presence)

- **Indicator:** Living dot particle field — multiple small dots in greens that orbit, breathe, and pulse. Not a static circle, not an avatar, not a face. The metaphor is organic growth — the system is alive.
- **Dot colours:** `['#059669', '#059669', '#047857', '#3D5A48', '#10B981']` — forest to emerald range, weighted toward emerald for energy
- **Rendered via:** `<canvas>` elements with `requestAnimationFrame`, `IntersectionObserver` for performance
- **Static fallback:** Fixed dot positions for `prefers-reduced-motion`
- **Typing state:** Dot field increases activity/brightness
- **Prompt input:** Expanding textarea in capsule container, bottom of workspace with generous padding (48px bottom). DM Sans at `text-base`.
- **Personality in copy:** Warm but not cute. Confident but not assertive. "Here's what I found" not "I'd love to help you with that!"

### Conversation Layout

- **Centred content:** `max-width: 720px`, centred like Claude/ChatGPT
- **No avatars:** No profile pictures or icons next to messages
- **No sender labels:** No "Ditto" or "You" text — the visual treatment distinguishes
- **User messages:** Subtle surface background pill with border-radius
- **Ditto messages:** No background, flows directly as typography
- **Content hierarchy:** Created through typography, whitespace, and accent borders — not cards
  - Synthesis blocks: `border-left: 2px solid var(--vivid)` (emerald) + padding-left
  - "Never" rules / emphasis: `border-left: 2px solid var(--vivid-deep)` (forest)
  - Hero moments (process, connect): surface background with border-radius
  - Guided options: pill buttons with border

## Layout

### Three-Column Workspace (Desktop 1024px+)

```
┌──────────┬───────────────────────────────┬────────────┐
│ Sidebar  │        Centre Content         │   Ditto    │
│ 240px    │        flex (min 480px)       │   Panel    │
│          │                               │   320px    │
│ Today    │  [Daily Brief / Feed /       │            │
│ Inbox 3  │   Review / Detail /          │            │
│ Work     │   Conversation]              │            │
│ Projects │                               │            │
│ Routines │  ┌─────────────────────────┐ │            │
│          │  │ Input bar (sticky btm) │ │            │
│ Settings │  └─────────────────────────┘ │            │
└──────────┴───────────────────────────────┴────────────┘
```

- Sidebar: 240px fixed, primary navigation. Icon rail on medium screens (1024-1279px), hamburger on mobile (<1024px).
- Centre: the workspace — content centres within available space
- Ditto panel: 320px, right side. Contextual content composed by the Self. Header shows "Ditto" with dot particle canvas.
- Input bar: 48px bottom padding (matches Claude placement), expands vertically with content
- Max content width: 720px (conversation), 580px (artefact output)

**Sidebar navigation labels** (user language, not system language — Insight-073):

| Label | What it shows | System concept it replaces |
|-------|---------------|---------------------------|
| **Today** | Daily brief, what needs attention | "Home" / "Dashboard" |
| **Inbox** | Review items, captures, alerts (badge count) | — (already user language) |
| **Work** | Active work items, progress | "Tasks" |
| **Projects** | Outcome containers | — (already user language) |
| **Routines** | What Ditto runs for you, health, configuration | "Processes" |
| **Settings** | AI connection, integrations, preferences | — (utility, acceptable) |

**Trust labels** (user language):

| User sees | System concept |
|-----------|---------------|
| Setting (imperative) | Badge (descriptive) | System concept |
|-----------|---------------|---------------|
| "Check everything" | "You review" | Manual / supervised tier |
| "Spot check" | "Spot check" | Spot-checked tier |
| "Let it run" | "Runs itself" | Autonomous tier |

### Accessibility Requirements

- WCAG 2.1 AA across all surfaces
- Contrast: 4.5:1 text, 3:1 UI elements
- Keyboard: full operability, tab order follows visual order
- Focus: 2px offset ring in accent colour on all interactive elements
- ARIA landmarks on major regions
- Screen reader labels on all interactive elements
- Functional at 200% browser zoom
- `prefers-reduced-motion`: static dot fields, no animation
