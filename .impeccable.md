# Impeccable Design Context — Ditto

This file persists Ditto's design DNA for all AI-assisted design and build work. Every `/frontend-design` invocation reads this first.

## Design Context

### Users

Ditto serves **outcome owners** — people who own business results but aren't developers. They need AI to handle execution so they can focus on judgment.

**Primary personas:**
- **Rob** — Trades MD, 8-15 staff, mobile between job sites. Approves quotes from his truck. Needs: glanceable, one-tap actions, nothing that feels like admin software.
- **Lisa** — Ecommerce MD, 10-20 staff. Manages content, customer service, warehouse ops. Needs: content quality consistency, proactive issue detection, confidence that things are running well.
- **Jordan** — Generalist technologist in 50-200 person org. Tech-savvy non-developer. Needs: prove value in 48 hours, show leadership the system map, be the leverage point.
- **Nadia** — Team manager, 5-10 specialists. Knowledge work requiring judgment. Needs: team's operational burden handled, focus on coaching not formatting.
- **Libby** — Doula transitioning to online education business. Building phase. Needs: knowledge capture, brand voice, social content creation. Primary prototype persona.

**Context of use:** These people enter work (questions, tasks, goals) into a living workspace. They are NOT managing automation flows or configuring pipelines. Ditto is where they work, not where they administer.

**Emotional journey:** Cautious Hope (Week 1) → Growing Trust (Month 1) → Confident Delegation (Month 2) → Compound Effect (Month 3+). The UI must support this arc — starting simple and intimate, growing structured and powerful.

### Brand Personality

**Three words:** Confident. Warm. Distinctive.

**Voice:** Like a competent colleague who remembers everything, surfaces what matters, and never makes you feel dumb. Not an assistant (too subservient), not a platform (too cold), not a chatbot (too casual).

**Visual identity philosophy:** "Precise White Lab" — a pristine canvas-white surface with sharp, functional typography, soft diffused elevation, and a single electric orange accent reserved for moments that need attention. The discipline is restraint: most surfaces are quiet; one signature color sparks. Inspired by Humble (humbleops.com), Linear, Raycast, and Framer — peer set for the operator audience Ditto serves.

**Emotional goals:**
- **Confidence** — "I can see what's happening and trust what I'm seeing"
- **Calm** — "Nothing is screaming at me; the important things surface naturally"
- **Competence** — "This system knows what it's doing"
- **Continuity** — "It remembers. It learns. It doesn't start from scratch."

**Anti-goals:**
- Never feel like Monday.com or Notion (passive containers for tasks)
- Never feel grey/dashboard-utilitarian (no dense data tables, no grey-on-grey dashboards)
- Never feel like a chatbot wrapper (artefacts are primary, chat is secondary)
- Never feel noisy (oversight is quiet, trust-calibrated, exception-driven)
- Never feel like a Claude/ChatGPT clone (no terracotta, no purple, no generic AI aesthetic)
- Never lean on heavy shadows, hard borders, or aggressive color blocking — clarity through soft elevation, restraint through single-accent discipline

### Aesthetic Direction

**Visual tone:** Precise and restrained with a single point of energy. Pristine, not sterile. Confident, not loud. Technical, not cold. Distinctive through discipline, not decoration.

**References (what to channel):**
- Humble (humbleops.com) — Precise White Lab aesthetic, single orange accent, soft diffused elevation
- Linear — minimalist functional UI, subtle shadows, precise typography
- Raycast — bright surfaces with sharp accent moments, clarity-first
- Framer — airy composition, clean product shots, intentional rhythm
- Ditto distinguishes with **electric orange** as its single accent (`#ff4000`), **Bricolage Grotesque** display + **Geist** UI typography, and **living dot particles** (ink black with orange spark)

**Anti-references (what to avoid):**
- Retool / internal tools aesthetic (dense, utilitarian, grey)
- Slack (noisy, notification-heavy, overwhelming)
- Enterprise dashboards (charts everywhere, tiny text, no breathing room)
- Generic AI chat apps (dark sidebar, white chat panel, blue/purple accents, Inter font)
- Claude's terracotta — explicitly rejected as derivative
- Heavy drop shadows, hard borders, multi-color CTAs — anything that anchors instead of lifts

**Theme:** Light mode primary (canvas white `#fafafa`). Dark mode supported — ink black `#0e0e0e` canvas, canvas-white text, electric orange retained as signature accent.

**Colors to avoid:** Green (was Ditto's old identity, retired in the rebrand to avoid green's "growth/eco" cliché — kept only for semantic positive states), purple/violet (generic AI), blue (ubiquitous in tech), terracotta (Claude). Red only for semantic negative/error states.

### Design Principles

1. **Artefacts are primary, conversation is secondary.** The workspace shows what was produced, what needs review, what's changing. Chat is the seed that creates artefacts, not the primary surface. (Insight-080)

2. **Progressive disclosure, not progressive complexity.** Start with the essential. Reveal depth on interaction. The user should never see more than they need at any moment — but everything should be reachable. "Boiling frog" — complexity grows so gradually the user never notices. (human-layer.md)

3. **User language, not system language.** No "Goal," "Trust tier," "Process," "Agent" in labels. Use the words the user would use: "How it's going," "What needs you," "What I know about your business." (Insight-073)

4. **Silence is a feature.** When things are running well, the UI is calm. Autonomous processes appear as one-line digests, not item-by-item noise. The workspace gets quieter as trust grows. (ADR-011)

5. **Every data point has provenance.** Outputs show what knowledge was used ("based on"). Decisions show what evidence supported them. Trust shows what earned it. Progressive disclosure: summary → detail on click. (Insight-087)

6. **Content flows typographically; cards are deliberate moments.** Most content lives in flow — hierarchy through Bricolage/Geist typography, whitespace, and the orange-accent border-left for emphasis. The 30px-radius card with soft six-stop elevation is reserved for *structured* moments (forms, decision panels, hero callouts) — not slapped around every paragraph. Restraint is the system. (P08a v6 design evolution + Precise White Lab)

7. **Bold identity through restraint.** Three signature elements appear consistently: **Bricolage Grotesque** display + **Geist** UI typography, the **single electric orange** accent (`#ff4000`), and the **dot particle field** with its lone orange spark. The visual metaphor is *precise activity* — a focused, professional system that does one thing exceptionally well. Identity comes from discipline, not decoration.

### Background Treatment

Pristine canvas-white surface. No ribbon, no patterns, no gradients. The brand identity lives in **typography (Bricolage + Geist), single-orange accent discipline, and the dot particle field** — not background decoration. Sections lift through soft six-stop diffused shadows, never through borders or color blocks. The white canvas is the foundation; everything else is restraint and one accent.

### Typography

| Scale | Size | Line Height | Letter Spacing | Weight | Usage |
|-------|------|-------------|----------------|--------|-------|
| `text-xs` | 12px | 1.4 | — | 400/500 | Caption, metadata, timestamps |
| `text-sm` | 14px | 1.4 | -0.007em | 400/500 | Body, sidebar, UI labels |
| `text-base` | 16px | 1.4 | -0.02em | 400/500 | Body-lg, conversation lead |
| `text-lg` | 18px | 1.4 | -0.04em | 500 | Subheading |
| `text-xl` | 24px | 1.2 | -0.02em | 500/600 | Heading-sm, page section |
| `text-2xl` | 32px | 1.15 | -0.02em | 500/600 | Heading, page titles |
| `text-3xl` | 42px | 1.05 | -0.03em | 600 | Heading-lg, hero subtitles |
| `text-4xl` | 58px | 1.0 | -0.052em | 600/700 | Display, hero only |

- **Display font:** Bricolage Grotesque — primary heading font. Generous x-height, slightly condensed, modern. Tight letter-spacing especially at large sizes.
- **UI/body font:** Geist — secondary sans-serif for body, labels, navigation, code. Clean technical aesthetic, complements Bricolage.
- **Mono font:** Geist Mono (preferred), JetBrains Mono fallback — code blocks only.
- **Hierarchy rule:** All headings use Bricolage with weight 500 or 600. Body uses Geist 400 or 500. Never use Geist for display sizes ≥24px; never use Bricolage for body ≤16px.

### Shadows

Soft, diffused, light — Humble's six-stop layered elevation. Lifts elements gently rather than anchoring with heavy forms. Light-mode uses near-black at very low opacity (0.03–0.04) layered across six stops.

- `subtle`: `0 1px 2px rgba(0,0,0,0.02)` — hairline lift, hover states
- `medium`: six-stop diffused, `rgba(0,0,0,0.03)` peaking at `0 30px 30px -2.5px` — primary card elevation
- `large`: same six-stop pattern at `rgba(0,0,0,0.04)` peaking at `0 40px 40px -3px` — modals, hero callouts

**Rule:** Never use harsh single-stop shadows or heavy opacity. Elevation is achieved through diffusion, not weight.

### Motion

Responsive, not performative. Every animation serves a purpose.

| Type | Duration | Easing |
|------|----------|--------|
| Micro (hover, press) | 100-150ms | `ease-out` |
| Standard (panels, cards) | 200-250ms | `ease-in-out` |
| Entrance | 300ms | `ease-out` |
| Exit | 200ms | `ease-in` |

**Never exceeds 400ms.** No bouncing, spinning loaders, parallax, confetti, or blocking animations. Respect `prefers-reduced-motion`.

### Icons

Lucide React. 20px default, 16px in compact contexts. `stroke-width: 1.5`. Icons accompany text — never standalone without a label (except universally understood: close, menu, search).

### The Self (Ditto's Presence)

- **Indicator:** Living dot particle field — small dots in ink-black with a single electric-orange spark, orbiting, breathing, pulsing. Not a static circle, not an avatar, not a face. The metaphor is **precise activity** — the system is working, focused, alive.
- **Dot colours:** `['#1c1c1c', '#1c1c1c', '#6e6e6e', '#ff4000', '#1c1c1c']` — obsidian/granite anchors with one orange particle as the visual heartbeat
- **Rendered via:** `<canvas>` elements with `requestAnimationFrame`, `IntersectionObserver` for performance
- **Static fallback:** Fixed dot positions for `prefers-reduced-motion`
- **Typing state:** Dot field increases activity/brightness; orange particle's pulse intensifies
- **Prompt input:** Expanding textarea in capsule container (`rounded-3xl` for multi-row, `rounded-full` for single-line), bottom of workspace with 48px bottom padding. Geist at `text-base`.
- **Personality in copy:** Precise but never cold. Confident but not assertive. "Here's what I found" not "I'd love to help you with that!" Mirrors Humble's voice: "Most folks ramp in ~15 minutes" — authentic, specific, no hype.

### Conversation Layout

- **Centred content:** `max-width: 720px`, centred like Claude/ChatGPT
- **No avatars:** No profile pictures or icons next to messages
- **No sender labels:** No "Ditto" or "You" text — the visual treatment distinguishes
- **User messages:** Subtle surface background pill with border-radius
- **Ditto messages:** No background, flows directly as typography
- **Content hierarchy:** Created through typography, whitespace, and accent borders — not cards
  - Synthesis blocks: `border-left: 2px solid var(--vivid)` (electric orange) + padding-left
  - "Never" rules / emphasis: `border-left: 2px solid var(--vivid-deep)` (deep orange)
  - Hero moments (process, connect): surface-raised background with 30px border-radius and `--shadow-medium`
  - Guided options: pill buttons (`rounded-full`) with hairline border

- Sidebar: 240px fixed, primary navigation. Icon rail on medium screens (1024-1279px), hamburger on mobile (<1024px).

- Max content width: 720px (conversation), 580px (artefact output)

### Artifact Mode (artifact open for editing/review)

When an artifact is open, the layout transitions gracefully. The sidebar **collapses** (icon rail at 56px or fully hidden). The conversation slides left into the freed space. The artifact takes the dominant centre. The Ditto context panel **stays put** — it never disappears.

```
┌──────────────┬──────────────────────────────┬────────────┐
│ Conversation │         Artifact             │   Ditto    │
│   300px      │      centre (flex)           │  Context   │
│              │                              │   320px    │
│ [refinement  │  [document / spreadsheet /  │            │
│  thread]     │   image / live preview /    │ [knowledge │
│              │   email / pdf]              │  used,     │
│              │                              │  process   │
│              │                              │  context,  │
│              │  ┌────────────────────────┐ │  provenance│
│ ┌──────────┐ │  │ provenance + actions  │ │  citations]│
│ │ input    │ │  └────────────────────────┘ │            │
│ └──────────┘ │                              │            │
└──────────────┴──────────────────────────────┴────────────┘
     Sidebar collapsed (icon rail or hidden)
```

**Column specs:**
- Conversation: **300px fixed**. Compact — messages during refinement are short instructions ("use Q4 rates", "punchier caption"). Input bar at bottom.
- Artifact: **flex (min 480px)**. The dominant surface. Six viewer types: Document (markdown/HTML), Spreadsheet (interactive table with CRUD), Image (gallery/carousel), Live Preview (sandboxed iframe — dashboards, forms, presentations), Email (mail-client preview), PDF (page-faithful via PDF.js).
- Ditto Context: **320px fixed, collapsible**. Defaults to open. User can dismiss (chevron/collapse icon) for wider artifact editing. Shows knowledge citations, process context, related items, version history.

**The transition is graceful:** physically it's still 3 visible columns. The content shifts — sidebar content (nav) swaps for conversation content, and what was the centre (conversation) becomes the centre (artifact). The right panel stays identical.

**Context panel content in artifact mode:**
- **Knowledge used** — "Based on: PlaceMakers prices (Mar 25), Henderson specs, your margin rules"
- **Process context** — which process produced this, what step, what trust level
- **Related items** — other artifacts from same process run
- **Version history** — v1 → v2 → v3 with change summaries (compact list, not full version bar)
- **Teach-this prompts** — when the system spots a learnable pattern

**Collapse behaviour:**

| Viewport | Sidebar | Conversation | Artifact | Context |
|----------|---------|-------------|----------|---------|
| ≥1440px | Icon rail (56px) | 300px | flex | 320px |
| 1280-1439px | Hidden | 300px | flex | 320px |
| 1024-1279px | Hidden | 280px | flex | 280px |
| <1024px | Hidden | Hidden (swipe) | Full width | Hidden (swipe) |

Mobile: full-screen artifact with swipe-left for conversation, swipe-right for context. Bottom sheet for actions.

### Accessibility Requirements

- WCAG 2.1 AA across all surfaces
- Contrast: 4.5:1 text, 3:1 UI elements
- Keyboard: full operability, tab order follows visual order
- Focus: 2px offset ring in accent colour on all interactive elements
- ARIA landmarks on major regions
- Screen reader labels on all interactive elements
- Functional at 200% browser zoom
- `prefers-reduced-motion`: static dot fields, no animation
